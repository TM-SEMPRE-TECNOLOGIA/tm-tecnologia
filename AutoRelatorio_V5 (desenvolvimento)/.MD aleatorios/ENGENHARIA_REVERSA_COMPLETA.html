Regras de Negócio Implícitas

Extraídas por engenharia reversa do código — nunca foram documentadas formalmente.

RN-01
9 Contratos fixos com templates específicos
Os contratos são fixos (0908, 1507, 1565, 2056, 2057, 2626, 2627, 3575, 6122). Cada um tem um .docx template exclusivo em contracts/cXXXX/template/. Para adicionar: criar módulo + registrar em registry.py.
RN-02
Sort de pastas em 3 camadas (Tradicional)
1º pastas com "vista ampla" no nome. 2º pastas com número no início (sort numérico, não alfanumérico). 3º pastas com "detalhes". Demais: alfabético. NUNCA usar sort() padrão do OS.
RN-03
Formato de medidas no nome do arquivo (SP)
"01 - 3,10 x 2,95 - Desconto 1,89m².jpg" → Largura: 3.10m, Altura: 2.95m, Desconto: 1.89m², Subtotal: 9.15m², Total: 7.26m². Vírgula como separador decimal (padrão BR).
RN-04
Altura padrão de imagem = 7cm
Todas as imagens são inseridas com altura máxima de 7cm. Ajustado de 6cm (V3) para 7cm (V4+). Layout inteligente: portrait = 2 colunas, landscape = 1 coluna (largura total ~10cm).
RN-05
7 Placeholders dinâmicos no template
{{nr_os}}, {{data_elaboracao}} (auto hoje), {{data_atendimento}}, {{agencia_codigo}}, {{agencia_nome}}, {{endereco}}, {{responsavel_dependencia}}. Importante: o marcador {{start_here}} define onde inserir as fotos.
RN-06
4 Descrições padrão de serviço
Desc 1–4 são variações do texto "levantamento preventivo". Campos [Nome do Gerente] e [Número da Matrícula] eram substituídos manualmente no passado. No V3.2+ entram como placeholders dinâmicos.
RN-07
Contrato 1565 = exclusivamente SP2
São José do Rio Preto usa template diferente com SINAPI (referência de preços). Tem croquis técnicos, tabelas de cálculo complexas, múltiplas paredes por serviço, fator faces (×2 área).
RN-08
Fator Faces no SP2
Arquivo com "Faces 2" no nome tem área multiplicada por 2. "Faces 1" = normal. Default = 1 quando não especificado. Exemplo: "01 - 3,10 x 2,95 - Faces 2.jpg" → área × 2.
RN-09
Croquis técnicos SP2
Arquivos com "CROQUI" no nome são tratados diferente: inseridos antes das fotos do serviço, com legenda extraída do próprio nome. e_croqui() detecta, extrair_legenda_croqui() extrai.
RN-10
Output sempre no backend/output/
Relatório gerado vai para backend/output/Relatorio-OS{nr_os}-{ag_cod}.docx. Nunca sobrescreve: se o arquivo existe, o novo é gerado e o antigo perdido. TODO: versionamento.
RN-11
Formato BR em toda a UI
Vírgula como separador decimal (3,10 não 3.10). Ponto como separador de milhar quando necessário. Datas em dd/mm/yyyy. Todas as formatações de float usam f"{v:.2f}".replace('.', ',').
RN-12
conteudo[] como contrato de dados
Qualquer scanner que produza um array conteudo[] no formato documentado pode ser consumido por qualquer builder. É o único "contrato" entre as duas camadas. Nunca quebrar esse contrato.
