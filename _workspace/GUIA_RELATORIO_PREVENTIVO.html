📐 Regras de Negócio Críticas
RN-01
Contrato 1565 = EXCLUSIVO SP2. Nunca tratar como Tradicional. São José do Rio Preto.
RN-02
Sort das fotos: "vista ampla" → numérico → alfabético → "detalhes". Nunca alterar essa ordem.
RN-03
Altura padrão das imagens: 10cm (Tradicional/SP). 7cm = exclusivo SP2 (contrato 1565).
RN-04
conteudo[] = único contrato de dados entre scanner e word_builder. Não criar alternativas.
RN-05
"Faces 2" no nome do arquivo = área × 2. Exclusivo SP2 (contrato 1565).
RN-06
Placeholders são case-sensitive: {{ag_cod}} ≠ {{AG_COD}}. Usar lista oficial.
RN-07
Arquitetura: 1 contrato = 1 módulo Python isolado. Nunca mudar o core por causa de 1 contrato.
RN-08
Nunca inventar itens. Se código não existe no items.json, reportar ao operador.
H01 — Heurística Validada
Nunca confiar no sort do SO. Windows e Python têm comportamentos diferentes — usar sort_key() customizado.
H03 — Heurística Validada
Detectar orientação da foto (portrait/landscape) pelo tamanho real, não pelo nome do arquivo.
H05 — Heurística Validada
Testar sempre com fixture mínima (2-3 fotos) antes de rodar com pasta de produção.
D02 — Descoberta Técnica
Regra "Faces 2": arquivo com "Faces 2" no nome → área × 2. Ex: 5,00×2,00 + Faces 2 = 20m².
🔧 Erros Comuns e Como Resolver

TS01 — .docx gerado com conteúdo só no cabeçalho

Causa: root_path não está sendo passado corretamente ao scanner.
Solução: Verificar se {{start_here}} existe no template. Está relacionado ao Sprint 3 — endpoint de upload de fotos.

TS02 — Foto com orientação errada no Word

Causa: Orientação detectada pelo tamanho nominal, mas EXIF pode ter rotação aplicada.
Solução: Usar Pillow Image.open().rotate() com exif_transpose=True antes de inserir a foto no Word.

TS03 — Sort de fotos em ordem errada

Causa: Sort usando os.listdir() que respeita case do Windows.
Solução: Usar a função sort_key() documentada em PADROES_TECNICOS.md §1.4.

TS04 — Placeholder não substituído no Word

Causa: Placeholder no template tem espaços ou quebra de linha interna no XML do .docx.
Solução: Rodar fix_xml.py para normalizar o XML antes de processar. Sempre executar após gerar_memorial.py.

TS05 — Engine do contrato não encontrado

Causa: Import no registry.py faltando ou com caminho errado.
Solução: Verificar backend/core/registry.py — todos os 9 contratos devem estar registrados.

Divergência de total no memorial vs corpo

Causa: Item aparece múltiplas vezes no corpo e a consolidação somou diferente.
Solução: Reportar ao Claude com o código e os valores divergentes. O script verificar_divergencias.py identifica automaticamente (tolerância: 0,01).
